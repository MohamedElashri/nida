# Writing for the Open Web

One of the nicest properties of a static site is that the final artifact is easy to inspect.

You can open the generated files, read the markup, upload them to simple hosting, and keep the publishing pipeline understandable years later. That durability is part of the appeal.

Nida leans into that by treating the generated output directory as the canonical served artifact set.
